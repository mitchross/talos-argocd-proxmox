# Talos 1.13 OSS NVIDIA Migration Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the Talos 1.13 GPU worker from proprietary NVIDIA kernel modules to NVIDIA open kernel modules while keeping GPU scheduling and llama.cpp service availability recoverable.

**Architecture:** Talos continues to own the host driver and NVIDIA container toolkit through system extensions; the NVIDIA GPU Operator remains limited to device plugin, GFD, validator, and runtime class management. Because this cluster currently has a single GPU worker, treat the rollout as a maintenance-window migration with explicit rollback rather than a true canary.

**Tech Stack:** Talos 1.13 beta, Omni/omnictl, Proxmox provider, NVIDIA GPU Operator v25.10.1, Argo CD, Kubernetes v1.35.4, llama.cpp

---

## Self-Review Notes

- The plan intentionally follows the Talos 1.13 beta docs rather than stable docs because the cluster is already on `v1.13.0-rc.0`.
- The largest migration uncertainty is the OSS driver validator path; the plan changes it to `/usr/local/glibc/usr/lib` because that is what both Talos 1.13 beta GPU guides currently document.
- Because there is only one GPU worker, this is not a real canary rollout. Treat it as a controlled maintenance window with fast rollback.
